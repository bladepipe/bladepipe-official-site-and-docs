本文档主要介绍 CloudDM Team 中数据库 CI/CD 服务配置，包含 CI/CD 服务和 IM 服务。

## CI/CD 服务

### 添加 CI/CD 服务 {#add_scm}

1. 登录 CloudDM Team 控制台。
2. 进入 **配置** > **CI/CD** 菜单。
3. 点击右上角 **添加服务提供者**，在弹出框中填写 CI/CD 提供者信息。
4. 点击 **测试**，CloudDM Team 会使用填写的信息进行 API 可用性测试。
5. 测试通过后点击底部 **添加** 按钮添加服务。

### 修改 CI/CD 服务

1. 登录 CloudDM Team 控制台。
2. 进入 **配置** > **CI/CD** 菜单。所有已配置的 CI/CD 服务都在列表中展示。
3. 在需要修改的项目上，点击最右侧的 **配置** 按钮，进入编辑模式。编辑模式下已经配置过的 CI/CD AccessToken 属性不显示。
4. 更新信息后点击底部 **保存** 按钮保存配置。

### 删除 CI/CD 服务

1. 登录 CloudDM Team 控制台。
2. 进入 **配置** > **CI/CD** 菜单。所有已配置的 CI/CD 服务都在列表中展示。
3. 在需要修改的项目上，点击最右侧的 **配置** 按钮，进入编辑模式。
4. 点击底部 **删除** 按钮删除 CI/CD 服务。

## IM 服务

IM 服务是通过 WebHook 方式调用 IM 服务提供商，将变更状态以消息形式通知用户，方便用户随时掌握最新消息。

### 添加 IM 服务 {#add_im}

1. 登录 CloudDM Team 控制台。
2. 进入 **配置** > **IM** 菜单。
3. 点击右上角 **添加服务提供者**，在弹出框中填写 IM 提供者信息。
4. 点击 **测试**，CloudDM Team 会使用填写的信息进行 API 可用性测试。
5. 测试通过后点击底部 **添加** 按钮添加服务。

### 修改 IM 服务

1. 登录 CloudDM Team 控制台。
2. 进入 **配置** > **IM** 菜单。所有已配置的 IM 服务都在列表中展示。
3. 在需要修改的项目上，点击最右侧的 **配置** 按钮，进入编辑模式。编辑模式下已经配置过的 IM AccessToken 属性不显示。
4. 更新信息后点击底部 **保存** 按钮保存配置。

### 删除 IM 服务

1. 登录 CloudDM Team 控制台。
2. 进入 **配置** > **IM** 菜单。所有已配置的 IM 服务都在列表中展示。
3. 在需要修改的项目上，点击最右侧的 **配置** 按钮，进入编辑模式。
4. 点击底部 **删除** 按钮删除 IM 服务。

## 存储空间

CloudDM Team 数据库 CI/CD 功能在使用时会根据发布流的配置，下载用户的源码仓库。   
默认情况下仓库下载目录位于 `APP_HOME/data` 目录下 `default` 和 `temporary` 两个子目录中。

管理员可以在 **设置** > **系统偏好** 页面中 **CloudDM** 选项卡下，通过下列属性修改存储路径：
- **defaultProjectSpace**：默认下载存储空间位置（可以配置 **绝对路径** 或 **相对路径**）
- **defaultTempSpace**：默认临时文件存储空间位置（可以配置 **绝对路径** 或 **相对路径**）
