本文主要介绍如何管理 CloudDM Team 中已添加的数据源，包括修改地址、账号等。

## 查看数据源

1. 点击 **数据源管理**，进入数据源列表页面。
2. 数据源列表展示了已添加的所有数据源，支持按类型、数据源 ID、数据源数字 ID、数据源描述、部署类型、网络类型和 host 地址搜索数据源。

## 修改数据源地址

已添加的数据源若想修改内网或外网地址，可以在数据源管理页面进行修改。

1. 选择需要修改的数据源，在右侧操作栏点击 **更多**。
2. 点击 **修改内网地址** 或 **修改外网地址**。
3. 在对话框中输入修改后的地址，点击 **确定**。

## 修改数据源账号

1. 选择需要修改的数据源，在右侧操作栏点击 **更多**。
2. 点击 **修改账号**。
3. 在对话框中选择认证方式，并填入相应信息后，点击 **保存**。

## 删除数据源账号

1. 选择需要删除账号的数据源，点击 **更多**。
2. 点击 **删除账号**。
3. 在确认删除的对话框中点击 **确定**。

## 删除数据源

1. 选择需要删除的数据源，点击 **更多**。
2. 点击 **删除**。
3. 在对话框中输入 **DELETE_DATASOURCE**，点击 **确定**。

:::info
如工单或数据管理正在使用数据源，则该数据源无法删除。需全部关闭后方可删除。
:::

## 修改阿里云 AK/SK

1. 选择需要修改的阿里云数据源，点击 **更多**。
2. 点击 **修改阿里云 AK / SK**。
3. 出现修改阿里云 AK / SK 的弹窗，输入相应的信息，点击 **保存**。

## 修改参数设置

1. 登录 CloudDM Team 控制台。
2. 点击 **查询设置**。
3. 选择数据源，点击操作一栏的修改参数配置。
4. 在修改后的参数值中填入配置后点击生效配置。
5. 检查修改的参数是否正确，点击确认。
6. 刷新参数配置，若参数当前运行值为刚才配置的值，则说明 **修改参数设置成功**。





## 启用查询功能 {#query}

1. 登录 CloudDM 控制台。
2. 点击 **查询设置**。
3. 选择数据源，点击数据管理的开关后。
4. 选择绑定的集群后，点击确认。
5. 返回数据源列表中，选择的数据源数据管理为 **开启状态** 则说明启用查询功能成功。

## 启用 CI/CD 功能 {#cicd}

1. 登录 CloudDM 控制台。
2. 点击 **查询设置**。
3. 选择数据源，点击 CI/CD 的开关决定这个数据源是否可以在 CI/CD 中使用。
4. 返回数据源列表，选择的数据源 CI/CD 为 **开启状态** 则说明启用成功。


## 添加数据源

目前支持的数据库为：MySQL、MariaDB、PostgreSQL、IBM DB2（IBM i 及 z/OS）、Greenplum、OceanBase(MySQL)、PolarDB-X、PolarDB(MySQL)、TiDB、SQL Server、阿里云 AnalyticDB(MySQL)、阿里云 RDS(MySQL)、阿里云 RDS(PostgreSQL)、阿里云 PolarDB(MySQL)、阿里云 PolarDB-X、亚马逊 AWS MySQL、亚马逊 AWS PostgreSQL、微软 Azure(MySQL)、微软 Azure(PostgreSQL)。

1. 登录 CloudDM Team 控制台。
2. 点击 **数据源管理** > **新增数据源**。
3. **部署类型** 选择 **自建**，并完成以下配置。
    - 数据库类型：目前支持 19 种数据源，详细请看 **[产品功能](../../intro/product_func)**。
    - 环境：默认为 Test Environment，在添加数据源时可根据情况分配。
        - 管理员可通过 **[环境管理](../cluster_worker)** 页面管理环境。
    - 获取方式：默认为手动。
    - 网络地址：可选择 **内网** 或 **外网**，并输入连接地址。
    - 认证方式：选择相应的认证方式，并填写所需内容，如账号密码等。
    - 描述：填写便于识别的名称。
    - 物理位置：选择 **不限** 或根据实际情况选择（非强依赖属性）。
    - 额外参数：不同的数据源可设置不同的参数。鼠标放置在 ![](../assets/operation/dsmanager/1.png) 图标即可查看参数说明。
      如需修改，点击 **修改后的参数值** 一列中的 ![](../assets/operation/dsmanager/2.png) 图标，并输入相应参数值，点击 **确定**。
4. 点击 **新增数据源** 添加数据源。
5. 返回数据源列表中，该数据源的 **创建状态** 为 **创建完成**。

## 小贴士

DB2 数据源需要选择具体的驱动来访问您的数据库，如：
- 基于 JT400，使用 JTOpen 驱动来访问您的数据库，那么驱动需要选择 JTOpen，比如 **IBM i**。
- 在数据源添加后，也可通过 查询设置 > 查询配置 > 修改参数配置，修改 driverVersion 参数为 'JTOpen-21.0.0'，修改驱动为 JTOpen v21.0.0。

SQL Server 数据源需要访问具体的驱动来访问您的数据库，如：
- SQL Server 2000 需要使用 jTDS 驱动来访问您的数据库。
- CloudDM 暂不支持SQL Server 添加数据源时选择实例。



## 阿里云数据源

### 准备阿里云子账号
参考 [阿里云 RAM 访问控制](https://help.aliyun.com/product/28625.html?spm=5176.2020520153.help.dexternal.3820336anMrJ7b)
文档，创建一个供 CloudDM Team 访问和操作阿里云数据库的子账号，并生成对应的 AccessKey ID (AK) 和 AccessKey Secret (SK)。一般而言，CloudDM Team 添加阿里云数据源需要子账号具备的基本权限包括：
- 通过阿里云 OpenAPI 获取数据源列表。
- 通过阿里云 OpenAPI 添加数据源白名单权限。

### 配置阿里云 OpenAPI 调用使用的 AccessKey ID 和 AccessKey Secret

1. 点击 **系统设置** > **第三方配置**。
2. 输入有数据源 OpenAPI 调用权限的 AccessKey ID 和 AccessKey Secret。
3. 点击 **授权访问**，配置成功。
    - 若原本已授权过，则将覆盖原来的配置，更新成当前的配置。
4. 如果用户不希望 CloudDM Team 保存 AccessKey ID 和 AccessKey Secret，可在添加完数据源以后再次打开该窗口，点击 **解除访问权限** 来避免 CloudDM Team 存储该授权信息。

### 新增数据源
1. 点击 **数据源管理** > **新增数据源**。
2. **部署类型** 选择 **阿里云**，并完成以下配置。
    - 数据库类型：选择希望添加的数据源类型
    - 地区：根据实际情况选择（非强依赖属性）。
    - 环境：默认为 Test Environment，在添加数据源时可根据情况分配。
        - 管理员可通过 **[环境管理](../cluster_worker)** 页面管理环境。
    - 获取方式：AccessKey，如之前并未在 **第三方配置** 中配置 AccessKey ID 和 AccessKey Secret 将显示对话框要求输入相应信息。
3. 下方列表会展示子账号有权限查看的数据源列表。
4. 勾选左侧需要添加的数据源，点击 ![](../assets/operation/dsmanager/3.png) 图标，将数据源添加到右侧，确认无误后点击 **下一步**。
5. 选择 **数据库认证方式**，并输入账号、密码等信息。
6. 在 **阿里云 AccessKey ID / AccessKey Secret** 一列输入相应信息。
7. 点击 **新增数据源**，添加数据源。
8. 返回数据源列表中，该数据源的 **创建状态** 显示为 **创建完成**。
