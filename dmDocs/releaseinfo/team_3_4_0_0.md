- 发版时间: 2025 年 12 月 19 日
- 版本号: v3.4.0.0

## 更新亮点
- 支持超大结果集查询，并支持边查询边查看数据，有效提升应用响应速度。
- 官网文档全新改版，优化阅读体验。

## 新增
  - 支持 导出选中的行、当前页和全部数据。
  - 支持 [TGZ 下载并部署](../maintain/install_tgz)。

## 优化
  - 优化 分析查询计划时，结果集中会注明查询结果属于 “计划”。
  - 优化 双击表后出现的默认 SELECT 语句不再增加 limit 语句，系统会自动添加。
  - 优化 查询结果集列较多时，仍会有横向滚动条，方便查看其它列的信息。
  - 优化 选择结果集导出格式（Json、Excel、SQL）的位置，从菜单选择改为在导出页面中选择。
  - 优化 Tab 标签栏和结果集标签栏在 默认、悬停、当前 三个状态下的配色，使其更加协调。
  - 优化 新建 SQL 规范后，默认规则从全部启用改为默认不启用，由用户决定启用哪些规则。
  - 优化 查询结果集相关参数，所有参数值不再支持无限大。
  - 优化 MySQL 解析器，解析能力得到全面提升。
  - 优化 PostgreSQL、Oracle、StarRocks、Doris 解析器，解析能力增强。

## 修复
  - 修复 查询结果集顶部 SQL 提示部分的交互可能挡住关闭按钮的问题。
  - 修复 SQL 审计页面，当一条 SQL 中包含太多资源时表格布局被破坏的问题。
  - 修复 在禁用安全规范或者规范中的规则时，环境描述过长导致布局被破坏的问题。
  - 修复 在导出 500 条以上含有脱敏数据的结果集到 EXCEL 中时，单元格备注提示信息丢失问题。
  - 修复 在数据接收期间无法中断查询的问题。
  - 修复 当递交执行多条 SQL 时，无法中断执行的问题。
  - 修复 在多浏览器或多窗口协同操作过程中，错误地提示没有删除或访问数据源权限的问题。
  - 修复 数据源在禁用后，查询控制台窗口未同步禁用状态的问题。
  - 修复 结果集序号列和数据行可能出现错行的问题。
  - 修复 PostgreSQL 解析器不支持 rename 语句的问题。
  - 修复 Oracle 执行超时后，控制台状态无法恢复正常的问题。
  - 修复 不支持 PostgreSQL in 查询语句中携带 limit 的问题。
  - 修复 PostgreSQL 语句块无法执行的问题。
  - 修复 数据源已经删除但提示权限不足的问题。
  - 修复 在没有选中任何数据库视图的情况下缺失右键菜单的问题。
  - 修复 查询设置中环境分页使用了数据源的分页数据，造成环境分页逻辑错误的问题。
  - 修复 CI/CD 在有只读权限的情况下，权限校验不通过时弹出框闪退的问题。
