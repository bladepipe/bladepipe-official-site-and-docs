- 版本号: v2.0.0.0

## 更新亮点
CloudDM Team 首次大版本更新，升级重点如下：

1. 采用了全新的 SQL 审核架构，新的架构下可提供更加强大、精准的 SQL 规则检测。
2. 支持的数据源从 **5** 种增加到 **15** 种。
3. 内置规则增加到 **59** 条，并且 **97%** 以上的规则适用于多种数据源。
4. 安全性提升，查询语句的 SQL 解析更加严格。对于未知、未处理的情况，默认策略从允许改为拒绝并报错。
5. 新增钉钉、企业微信、飞书作为审批流程引擎。

:::info
在升级前建议您仔细阅读 **本文** 和 **1.x 到 2.x 版本升级说明(../faq/upgrade_to_2_x)**，充分了解升级的变化。
:::

## 新增
- 新增 支持 PostgreSQL 数据源。
- 新增 支持 Greenplum 数据源。
- 新增 支持 OceanBase for MySQL 数据源。
- 新增 支持 PolarDB-X 及 阿里云 PolarDB-X 数据源。
- 新增 支持 阿里云 ADB for MySQL 数据源。
- 新增 支持 阿里云 PolarDB for MySQL 数据源。
- 新增 支持 亚马逊 AWS MySQL 数据源。
- 新增 支持 亚马逊 AWS PostgreSQL 数据源。
- 新增 支持 微软 Azure MySQL 数据源。
- 新增 支持 微软 Azure PostgreSQL 数据源。
- 新增 内置查询规则增加到 54 条，其中绝大部分规则支持多种数据源。 
- 新增 支持钉钉、企业微信、飞书作为审批流程引擎。
- 新增 支持 PostgreSQL 数据源 `create schema ... create table` 语句的连体写法。
- 新增 支持 PostgreSQL 数据源 `create database ... owner` 语句。
- 新增 支持 PostgreSQL 数据源 `create database ... template` 语句。
- 新增 支持 PostgreSQL 数据源 `alter database ... allow_connections` 语句。
- 新增 支持 PostgreSQL 数据源 `alter database ... is_template` 语句。
- 新增 支持 PostgreSQL 数据源 `drop database ... force` 语句。
- 新增 支持 PostgreSQL 数据源 `drop schema ...` 多名称。
- 新增 支持 MySQL 数据源 `alter schema abc default collate utf8mb4_unicode_ci;` 语句。
- 新增 支持 MySQL 数据源 `create database abc collate utf8mb4_unicode_ci;` 语句。

## 优化
- 优化 查询语句的 SQL 解析更加严格。对于未知、未处理的情况，默认策略从允许改为拒绝并报错。
- 优化 配置规范中的规则时，如果规则处于禁用状态，则不会再进行风险提示。
- 优化 已有规则优化，详见 “1.x 到 2.x 版本升级说明(../faq/upgrade_to_2_x)”。

## 修复
- 修复 PostgreSQL 数据源，`create schema ...`、`alter schema ...`、`drop schema ...` 的语句中同时指定库时报错的问题。
- 修复 PostgreSQL 数据源 `insert ... with ... select` 语句解析异常的问题。
